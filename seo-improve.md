# SEO: рекомендации при редиректе `/` → языковые версии

Ниже собраны предложения с учетом того, что корень `/` продолжает редиректить пользователей на `/ru/`, `/en/`, `/ka/`, а приоритет — продвижение RU‑версии.

## Исходные условия
- Корень `/` используется как точка входа и редиректит по языку.
- Есть отдельные URL для языков: `/ru/`, `/en/`, `/ka/`.
- RU — приоритет, EN/KA — вторичные.

## Ключевые риски текущей схемы
- Авто‑редирект по языку может усложнять корректную индексацию всех языковых версий.
- Корень `/` с русским контентом может конкурировать с `/ru/` в выдаче.
- В `src/index.ejs` атрибут `lang="x-default"` некорректен для HTML (это значение предназначено для `hreflang`).
- В `src/index.ejs` canonical указывает на `/`, а `og:url` — на `/ru/`, что даёт смешанные сигналы.

## Рекомендации (без изменения схемы редиректов)
- Сделать `/` служебной страницей для ботов: добавить `noindex, follow`, чтобы основные сигналы аккумулировались на `/ru/`.
- Если `/` остаётся индексируемым, привести `og:url` и `canonical` к одному URL.
- Поставить корректный `lang` для корня: `ru` или `ru-GE`, если текст русский.
- Оставить `hreflang` на всех языковых страницах и указать `x-default` на URL, который реально должен быть «дефолтным» (либо `/`, либо `/ru/` — выбрать одну стратегию).
- Оставить явные ссылки на все языковые версии в шапке/футере и на ключевых страницах (простые `<a href>`, не JS‑редиректы).

## Контент и структура (приоритет RU)
- Создать отдельные RU‑страницы под основные категории работ и частые проблемы:
- Примеры: стиральные машины, посудомойки, плиты, кондиционеры.
- На каждой странице: симптомы, что входит в ремонт, сроки, гарантия, диапазон цен, CTA.
- Уникальные `title` и `description` для каждой страницы, без дублирования между языками.
- Усилить перелинковку: с главной на услуги, цены, контакты, и обратно.

## Локальные сигналы
- Добавить полный адрес и район обслуживания в видимый текст (RU‑страницы).
- Расширить `LocalBusiness` schema: адрес, регион, почтовый индекс, часы.
- Оформить и вести профиль в Google Business Profile и Яндекс Бизнес.

## Технические улучшения
- Проверить, чтобы sitemap содержал все языковые страницы (сейчас это так, важно не терять).
- Оставить 302 для языкового выбора (как сейчас) — это нормальная практика для персонализированных редиректов.
- Избегать покупки ссылок и «схем» ссылочного продвижения.

## Предлагаемая стратегия по умолчанию (RU‑фокус)
- Сделать `/ru/` основным ранжируемым URL для RU‑запросов.
- `/` оставить как редирект‑хаб и вывести его из индекса `noindex, follow`.
- В `hreflang` назначить `x-default` на `/ru/` или `/` и придерживаться выбранной логики везде.
