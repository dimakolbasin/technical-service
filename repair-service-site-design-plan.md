# Сайт-визитка сервиса ремонта бытовой техники в Батуми  
## Дизайн + фронтенд-план (static, SEO-first, без JS)

---

## 1. Цель проекта
Создать простой, быстрый и максимально SEO-дружелюбный сайт-визитку для сервиса ремонта бытовой техники в городе **Батуми** на **русском / грузинском / английском** языках.

Приоритеты:
- статический HTML + CSS
- идеальная семантика
- ноль бизнес-логики
- минимум зависимостей и ноль билдера
- доверительный “сервисный” дизайн
- масштабируемость под контент и языки

---

## 2. Технические принципы (скелет без логики)
- **Static-first**: чистый HTML + CSS, без JS (допустим только JSON-LD в `<head>` при необходимости).
- **Mobile-first**, затем расширения через media queries.
- **Единый `styles.css`**: весь CSS живёт в одном файле, никаких сборщиков или билд-процессов.
- **Без внешних зависимостей**, кроме шрифта, предпочтительно локально.
- **Одинаковая структура всех языков** и одинаковые классы BEM.

---

## 3. Стек
- HTML5 (семантика) + чистый CSS3
- CSS custom properties (design tokens)
- Flexbox + Grid
- BEM-нейминг
- SVG для иконок

---

## 4. Шрифт
**Noto Sans** — универсальная семейка для RU / KA / EN, меньше плясок с фолбэками.

Подключение:
- локально (`woff2`, `/assets/fonts/`)
- один файл на весь сайт, никаких вариаций по языкам

---

## 5. Общая концепция дизайна
**Стиль:** спокойный сервисный минимализм, доверительный и аккуратный.

**Визуальный язык:** светлый фон, мягкие поверхности, тонкие границы, аккуратные тени, простые line-иконки. Главная идея — быстро объяснить: «что ремонтируем / где / как быстро / гарантия / как связаться». 

---

## 6. Цветовая система
- `--c-bg`: почти белый фон страницы
- `--c-surface`: белый для карточек и модалей
- `--c-text`: близкий к чёрному, но не чистый
- `--c-muted`: нейтральный серый для вторичных подписей
- `--c-border`: тонкая светлая граница
- `--c-primary`: спокойный синий/бирюзовый акцент
- `--c-primary-contrast`: белый текст на акценте

**Правила:**
- акцент только для CTA, ссылок, активных элементов
- никаких градиентов и пестроты
- не более 2–3 цветов в одной зоне
---

## 7. Layout-скелет страниц (повторяемость = скорость)

### Общий шаблон
- header (логотип, телефон, язык)
- nav
- main
- footer

### Повторяемые компоненты
- section (универсальная)
- hero
- card
- button
- badge
- list
- form (даже если пока без отправки — визуальная часть)
- breadcrumbs (опционально)

---

## 8. Компонентный план (что верстать первым)

### 8.1 База (1-й этап)
- reset + tokens + base typography
- container + grid
- header
- nav (desktop + mobile без JS: либо всегда открытое меню, либо `<details>/<summary>`)
- footer

### 8.2 Главная (2-й этап)
- hero (H1, подзаголовок, CTA)
- services (карточки с ссылками)
- how-it-works (4 шага)
- pricing teaser (2–4 пункта)
- faq teaser (3–5 вопросов)
- contact-cta (липкая кнопка звонка на мобиле — CSS-only)

### 8.3 Страницы услуг (3-й этап)
- service intro (симптомы, бренды, гарантия)
- common problems (список)
- prices block
- faq по услуге

### 8.4 Контакты / цены (4-й этап)
- contact card (`tel:`, `mailto:`, мессенджеры)
- service area (районы списком)
- price table (адаптивная таблица / карточный вариант)

---

## 9. Адаптив и сетки (без усложнений)
- container: max-width 1100–1200px, padding 16px (mobile) / 24px (desktop)
- карточки: 1 (mobile) / 2 (tablet) / 3 (desktop)
- таблицы: на мобиле карточный вид или горизонтальный скролл

---

## 10. Доступность и качество
- интерактивы только `a` и `button`
- `:focus-visible` и видимые фокус-стили
- достаточный контраст
- меню через `<details>` без JS

---

## 11. Мультиязычность (frontend)
- папки `/ru`, `/en`, `/ka`
- одинаковый шаблон и классы BEM
- языковой переключатель — ссылки на те же страницы
- `<html lang="">` соответствует языку
- hreflang + языковые ссылки

---

## 12. Минимальный набор файлов
- `index.html`
- `styles.css`
- `assets/fonts/*.woff2`
- `assets/img/logo.svg` + 2–3 SVG-иконки

---

## 13. Чеклист готовности
- аккуратный вид на мобиле и десктопе
- нет JS (кроме JSON-LD в будущем)
- меню работает без JS
- единый header/footer и типографика
- чистый BEM
- одна дизайн-система, нет магии

---

## 14. Описание дизайна (законченная спецификация)

### 1) Общий стиль
- «спокойный сервисный минимализм»
- светлый фон, мягкие поверхности, тонкие границы
- иконки line, `currentColor`
- идея: быстро сказать «что, где, как, гарантия, связь»

### 2) Цветовая система
- акцентный цвет для CTA, ссылок и активных состояний
- `--c-bg`, `--c-surface`, `--c-text`, `--c-muted`, `--c-border`
- `--c-primary`, `--c-primary-contrast`
- никаких градиентов и избыточной палитры

### 3) Типографика
- Noto Sans для RU / KA / EN
- Base 16px, line-height 1.6
- H1 32–36px (desktop) / 28–32px (mobile)
- H2 24–28px, H3 18–20px, мелкий текст 14px
- `max-width: 65ch` и умеренный межстрочный ритм

### 4) Сетка и отступы
- container: 1120–1200px, padding 16px / 24px
- шаг отступов каждые 8px (4–8–16–24–32–48–64)
- карточки: radius 12px, border 1px, мягкая тень
- «воздушный» ритм, без декоративных излишеств

### 5) Компоненты
#### 5.1 Header
- логотип/название слева, телефон + часы справа
- языки как компактные ссылки
- тонкая линия снизу

#### 5.2 Навигация
- 4–6 пунктов (Услуги, Цены, Контакты, FAQ, О нас)
- mobile: `<details>` с “Меню”, активный пункт подчёркнут

#### 5.3 Hero
- H1 «Ремонт бытовой техники в Батуми»
- подзаголовок (выезд, гарантия, сроки)
- CTA: Primary «Позвонить», Secondary «Написать»
- иллюстрация или иконка справа (desktop)

#### 5.4 Преимущества
- 3–4 карточки с иконками: выезд, диагностика, гарантия, сроки

#### 5.5 Услуги
- карточки: заголовок, типовые проблемы, ссылка «Подробнее»
- hover: лёгкая тень / подъём (transition 150–200ms)

#### 5.6 Как работаем
- Заявка → Диагностика → Согласование → Ремонт + гарантия
- десктоп: линия, мобайл: столбик

#### 5.7 Цены
- preview на главной (3–5 строк)
- на странице цен — таблица, мобайл — карточки или скролл

#### 5.8 FAQ
- `<details>` для каждого вопроса
- ответы 2–5 строк

#### 5.9 Контакты
- контактная карточка: телефон, мессенджеры, часы, районы
- ссылка «Открыть на карте» без iframe
- мобайл: фиксированная кнопка «Позвонить» снизу

#### 5.10 Footer
- контакты, языки, копирайт, ссылки на политику

### 6) Иконки и графика
- только SVG line icons
- `currentColor`, одинаковая толщина
- не как декор, а как навигация взгляда

### 7) Состояния
- hover, focus-visible, active, disabled
- `details[open]` с аккуратным отступом и линией

### 8) Анимации
- только `transition` 150–200ms
- никаких параллаксов, каруселей

### 9) Тёмная тема
- не делаем на старте
- при необходимости — медиа-фича + токены

### 10) План задач
- определить токены цветов/типографики/отступов/радиусов
- собрать button, card, section, details
- непременно состояния hover/focus/active
- проверить контраст и читабельность на RU/KA/EN

---

## 15. Что не делаем
- SPA / React / Vue
- JS-карусели и сложные анимации
- iframe-карты и трекеры
- тёмную тему на старте

---

## 16. Следующий шаг
- генерация базового `index.html`
- начальный `styles.css` (токены, reset, header/nav/hero/cards/footer)
- никаких строк JS: всё работает через HTML + CSS + `<details>`

Если хочешь, могу следующим шагом выдать точный набор CSS-токенов и референсный шаблон главной страницы (HTML + CSS) без JS.
